# Security Policy

## Intended Vulnerabilities

This project is an **intentionally vulnerable** lab environment.
All services are configured insecurely by design for educational purposes.
The vulnerabilities listed in `README.md` are features, not bugs.

## Reporting Unintended Security Issues

If you find a security issue **in the lab infrastructure itself** (e.g., the
build pipeline, CI/CD workflow, or a secret accidentally committed), please
report it responsibly:

1. **Do NOT** open a public GitHub issue
2. Email: `security@your-domain.com` with subject `[MS3-ARM] Security Report`
3. Include: description, reproduction steps, potential impact
4. We will respond within 48 hours

## Ethical Use Policy

This lab MUST only be used:

- On isolated, private networks
- For personal learning, CTF practice, or authorized security training
- Never against production systems or without explicit written authorization

Misuse of this lab against systems you do not own is illegal and unethical.
